# RS DPM Planner

Desktop-first Angular application for planning manually defined RuneScape ranged
rotations and simulating theoretical ideal-condition damage output.

## Current Scope

- Ranged only
- Single target only
- No backend
- Strict validation
- Portable versioned import/export
- Manual JSON game data stored in the repo

## Tooling

- Angular 21
- TypeScript
- SCSS
- Vitest via Angular's test builder
- Playwright for E2E
- ESLint + Prettier

## Available Scripts

npm run dev

## Game Data Bundle

The app runtime loads a generated bundled catalog at
`public/game-data/catalog.sample.json` instead of fetching many individual
`src/game-data/**/*.sample.json` files.

- The bundle is generated by `scripts/generate-game-data-bundle.mjs`
- It is rebuilt automatically by `npm run dev`, `npm run build`, and `npm run test`
- The source of truth remains the split curated JSON files under `src/game-data/**`
- `public/game-data/catalog.sample.json` is generated output and is gitignored

If you add new curated game data, make sure it is included in
`src/game-data/loaders/sample-manifest.ts` so it becomes part of the generated
catalog.
